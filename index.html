Real Codebase Step-by-Step Setup Guide


Ready to trade the browser playground for the real heavy machinery? Great choice. Below is a friendly, no-nonsense walkthrough that gets Python installed, packages pulled in, the blockchain layer compiled, and both servers running locally — with tests to prove nothing broke along the way. Grab a coffee, open a terminal, and let's go one step at a time.

1

Install Python & Node.js

Two tools power everything here. Grab Python 3.10 or newer from python.org/downloads — that's the brain of the operation, running our FastAPI backend and calculation engine. Then grab Node.js LTS from nodejs.org, which handles compiling our Ethereum Hardhat smart contracts.

Windows users, don't skip this: during the Python installer, tick the checkbox that says "Add Python to PATH." Miss it, and your terminal will act like it's never heard of Python the moment you type a command — and we'll both be very sad about it.
Verify both installed correctly before moving on:
python --version
node --version
2

Install Python Libraries

Next, we feed the backend its toolkit — FastAPI to serve APIs at speed, SQLAlchemy to talk to the database, Streamlit to power the frontend panels, plus a handful of supporting libraries. It's best practice to do this inside a virtual environment so nothing clashes with packages elsewhere on your machine:

# create and activate a virtual environment (recommended)
python -m venv venv
source venv/bin/activate // macOS/Linux
venv\Scripts\activate // Windows

# install everything from the project root
pip install -r requirements.txt
If your terminal throws a wall of compiler errors, double-check that the venv is actually activated (you should see (venv) at the start of your prompt) before re-running the install. That alone fixes 90% of "it worked on my machine" issues.
3

Setup Solidity Contracts

Time to compile the blockchain layer. We use Hardhat to spin up a local Ethereum network simulation, and we lean on OpenZeppelin for battle-tested, audited security standards rather than reinventing token logic from scratch. Hop into the contracts folder and pull in the JavaScript packages:

cd contracts
npm install
cd ..
Want to double-check the contracts compile cleanly before moving on? Run npx hardhat compile from inside the contracts folder — a clean run means you're good to go.
4

Launch the Servers

Nobody wants to juggle 15 terminal tabs, so we didn't make you. A custom Python orchestrator script (launch.py) quietly handles the SQLite database migrations, seeds all the sample asset portfolios, compiles the Solidity files, and boots both the FastAPI backend and the Streamlit UI in parallel. One command, and everything comes alive:

python launch.py
Once your console shows both services reporting "online," open your browser to http://127.0.0.1:8501 to explore the institutional-grade Streamlit portal in person.
5

Run Code Tests

Before you trust any of the numbers on screen, let's make sure the math and the contracts actually hold up. Two quick test suites cover both sides of the stack — the Python calculation engine and the Solidity contract logic:

# run Python calculation verification tests
pytest tests/ -v

# run Solidity smart contract assertions
cd contracts
npx hardhat test

All green ticks? Congratulations — you're officially running a real, tested decentralized finance builder locally. If anything comes back red, re-read the error message closely; it almost always points straight at the missing dependency or misconfigured path.
